COOKIES | https://www.w3schools.com/js/js_cookies.asp
    * A cookie is a small piece of data that a website stores on your browser. 
    * helps websites “remember” things about you when you visit them again.
    * Cookies = “website memory” stored in your browser”.

    * Stores information
        * username or login session
        * Items in a shopping cart
        * User preferences (like theme color)
        * Cookies are stored locally, so the website can read them next time you visit.
        * Can have an expiration
        * Some cookies disappear when you close the browser (session cookies).
        * Others last days, months, or years (persistent cookies).
        * Used by websites for convenience
        * Keep you logged in
        * Remember your settings
        * Track your activity (for analytics or ads)

    * Example: Todo App
        * When you add a task to your todo app, the website can store it in a cookie.
        * if you refresh the page or close the browser, your tasks are still there when you come back.


    * document.cookie
        * JavaScript can create, read, and delete cookies with the document.cookie property.
    
        * Syntax    document.cookie = "username=John Doe";
        --
        * You can also add an expiry date (in UTC time). 
        * By default, the cookie is deleted when the browser is closed:

        * Syntax    document.cookie = "username=John Doe; expires=Thu, 18 Dec 2013 12:00:00 UTC";
        --
        *With a path parameter, you can tell the browser what path the cookie belongs to. 
        * By default, the cookie belongs to the current page.

        * Syntax    document.cookie = "username = John Doe; expires=Thu, 18 Dec 2013 12:00:00 UTC; path=/";

* JSON.stringify(todos) 
    * is a method that converts a JavaScript object or array
    * (in this case, the todos variable) into a JSON string.

    * Convert the list of todo items (probably stored as an array of objects) into a string format
    * Store this string representation in browser storage (like cookies or localStorage)

    * const todos = [
        { id: 1, text: "Buy groceries" },
        { id: 2, text: "Finish homework" }
    ];

    After applying JSON.stringify(todos), it becomes:
    "[{\"id\":1,\"text\":\"Buy groceries\"},{\"id\":2,\"text\":\"Finish homework\"}]"

try...catch?
    * way to handle errors in JavaScript so your code doesn’t break if something goes wrong.

    * try { ... } → contains the code that might throw an error.
    * catch (e) { ... } → runs if an error happens inside try. e contains information about the error.

    Example:
    try 
    {
        var data = JSON.parse("invalid JSON"); // This will fail
    } 
    catch (e) 
    {
        console.log("There was an error:", e);
    }

JSON (JavaScript Object Notation) 
    lightweight data interchange format that's easy for humans to read and write 
    and easy for machines to parse and generate.

    * A text-based format that is language independent
    * Derived from JavaScript object syntax, but distinct from it
    * Uses simple conventions that programmers are already familiar with

    Structure
        {
            "name": "John",
            "age": 30,
            "city": "New York"
        }
    Usage
        API Communication: Most web APIs send and receive data in JSON format
        Data Storage: Often used for configuration files and data persistence (like your todo list)
        Data Exchange: Transfers data between server and web applications

        JSON.stringify(): Converts JavaScript objects to JSON strings
        JSON.parse(): Converts JSON strings back to JavaScript objects

jQuery
    * JavaScript library
    * to make working with JavaScript easier, faster, and more consistent across browsers.

    * Simplifies DOM manipulation: 
        * Instead of writing long document.getElementById() or document.querySelector(), 
        * you can use a short and easy syntax like $('#myElement').

    * Cross-browser compatibility: 
        * It hides browser quirks (like old Internet Explorer issues) so your code works everywhere.

    * Event handling: 
        * Makes adding and removing event listeners shorter and easier.

    * Animations: 
        * Includes built-in effects like .fadeIn(), .slideUp(), etc.

    * AJAX requests: 
        * Makes it easier to load data from a server without reloading the page
